# i18n — стратегия локализации

## Стек

| Пакет                              | Роль                                         |
| ---------------------------------- | -------------------------------------------- |
| `i18next`                          | Ядро i18n                                    |
| `react-i18next`                    | Хуки `useTranslation`, компонент `<Trans>`   |
| `i18next-browser-languagedetector` | Автодетект языка из localStorage / navigator |

Конфиг: `src/shared/lib/i18n.ts`. Подключается в `main.tsx` до рендера приложения.

## Состояние MVP

- **Доступные локали:** только `ru`
- **Дефолт:** `ru`
- **Fallback:** `ru`
- Английская локаль будет добавлена post-MVP

## Структура локалей

**Сразу разбиваем на namespaces** по фичам (без промежуточного этапа единого файла — мы точно превысим 500 строк):

```
src/locales/ru/
  common.json         — кнопки, действия, общие слова
  nav.json            — пункты меню sidebar
  auth.json           — Login, Register, change password
  tasks.json          — TaskList, TaskDetail, статусы
  projects.json       — Repositories, NewRepository, WebhookSecretModal
  history.json        — лента публикаций
  analytics.json      — аналитика и графики
  dictionaries.json   — словари
  settings.json       — настройки профиля, GitHub, уведомлений
  onboarding.json     — onboarding модалка
  cmdk.json           — command palette
  errors.json         — backend error mappings + generic
```

В `i18next` конфиге:

```ts
resources: {
  ru: {
    common: commonRu,
    nav: navRu,
    auth: authRu,
    tasks: tasksRu,
    // ...
  },
},
ns: ['common', 'nav', 'auth', 'tasks', 'projects', 'history', 'analytics',
     'dictionaries', 'settings', 'onboarding', 'cmdk', 'errors'],
defaultNS: 'common',
```

В компонентах:

```tsx
const { t } = useTranslation('auth')
t('login_title') // namespace 'auth'
t('common:save') // явный namespace 'common'
```

## Конвенции использования

### В компоненте

```tsx
import { useTranslation } from 'react-i18next'

export function LoginForm() {
  const { t } = useTranslation()
  return <button>{t('auth.submit_login')}</button>
}
```

### Интерполяция

```json
{ "tasks": { "showing_count": "Показано {{shown}} из {{total}} задач" } }
```

```tsx
t('tasks.showing_count', { shown: 12, total: 247 })
// → "Показано 12 из 247 задач"
```

### Плюрализация

```json
{
  "tasks": {
    "selected_count_one": "{{count}} задача выбрана",
    "tasks_selected_count_few": "{{count}} задачи выбрано",
    "tasks_selected_count_many": "{{count}} задач выбрано"
  }
}
```

```tsx
t('tasks.selected_count', { count: 1 }) // "1 задача выбрана"
t('tasks.selected_count', { count: 3 }) // "3 задачи выбрано"
t('tasks.selected_count', { count: 12 }) // "12 задач выбрано"
```

### Богатый текст с компонентами

```tsx
import { Trans } from 'react-i18next'
;<Trans i18nKey="onboarding.connect_github">
  Привяжите <strong>GitHub-аккаунт</strong> чтобы продолжить
</Trans>
```

## Правила

1. **Никогда не хардкодить русский текст в JSX** — всегда `t('key')`
   - Линт-правило `i18next/no-literal-string` (добавить позже)
2. **Ключи — английские, snake_case с точкой как разделитель namespace**
   - `auth.login_title`, не `auth.loginTitle` или `авторизация.логин`
3. **Русский текст видно только в `ru.json`** — поиск по проекту по русской фразе сразу даёт ключ
4. **Сообщения ошибок бэка** — отдельный namespace `errors.backend.*`, маппинг в `shared/lib/errorMessages.ts`
5. **Даты и числа** — через `dayjs` (для дат) и `Intl.NumberFormat` (для чисел), не через i18next-форматтеры
6. **Код статусов** — `tasks.status.running`, чтобы повторно использовать в разных местах (badge, filter, header)

## Что НЕ переводим

- `file_path` — путь к файлу всегда в оригинале
- `commit_message` — git-сообщение пользователя
- `error.detail` от бэка для технических ошибок (логируем в Sentry)
- Email-адреса, GitHub-логины, SHA коммитов

## Маппинг бэкенд-ошибок (decisions.md, п.12)

В `src/shared/lib/errorMessages.ts`:

```ts
import i18n from '@/shared/lib/i18n'

const BACKEND_TO_KEY: Record<string, string> = {
  'Email already registered': 'auth.errors.email_taken',
  'Invalid credentials': 'auth.errors.invalid_credentials',
  'Not authenticated': 'errors.session_expired',
  'GitHub account is not linked': 'errors.github_not_linked',
  // ...
}

export function translateBackendError(detail: string): string {
  const key = BACKEND_TO_KEY[detail]
  if (!key) return i18n.t('errors.generic')
  return i18n.t(key)
}
```

Используется в RTK Query error handling middleware.

## Добавление новой локали (post-MVP)

1. Создать `src/locales/en.json` — копия `ru.json` с переведёнными значениями
2. В `src/shared/lib/i18n.ts` добавить:
   ```ts
   resources: {
     ru: { translation: ru },
     en: { translation: en },
   },
   supportedLngs: ['ru', 'en'],
   ```
3. Добавить переключатель языка в `/settings/profile` — dropdown «Язык интерфейса»
4. Запросить у дизайна светлую тему (если нет к этому моменту) — английская UI часто идёт со светлой темой по умолчанию для западных рынков

## Что отложено до post-MVP

- Английская локаль
- ESLint-правило для no-literal-string (после первой полноценной фичи)
- Серверная локализация писем, push-уведомлений
- Date/number форматирование с учётом locale (сейчас захардкожено `dayjs.locale('ru')`)
